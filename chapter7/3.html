函数重载与重载解析

函数重载与重载解析

  • 函数重载：使用相同的函数名定义多个函数,每个函数具有不同的参数列表
#include <iostream>

int fun(int x)
{
    return x + 1;
}

int fun(int x, int y)
{
    return x + y;
}

double fun(double x)
{
    return x + 1;
}

int main()
{
    std::cout << fun(3.5) << std::endl; // 4.5
    std::cout << fun(3) << std::endl; // 4
}
  1. 不能基于不同的返回类型进行重载
int fun(int x)
{
    return x + 1;
}
double fun(int x)
{
    return x + 1;
}
int main()
{
    fun(3); // 这样调用程序无法选择哪一个 fun 函数
}

名称查找
  • 限定查找( qualified lookup )与非限定查找( unqualified lookup )
  • 非限定查找会进行域的逐级查找 —— 名称隐藏( hiding )
#include <iostream>

void fun()
{
   std::cout << "golbal fun is called\n"; 
}

namespace MyNS
{
    void fun()
    {
        std::cout << "MyNS fun is called\n";
    }
    void g()
    {
        fun();
    }
}

int main()
{
    fun(); // golbal fun is called 非限定查找
    ::fun(); // golbal fun is called
    MyNS::fun(); // MyNS fun is called 限定在名字空间中查找
    MyNS::g(); // MyNS fun is called 非限定查找会进行域的逐级查找 —— 名称隐藏
}
  • 查找通常只会在已声明的名称集合中进行

下面的例子说明了这一点，编译是顺序执行的，当到 g() 函数时程序所能找到的 fun 函数只有第一个，如果在 g() 函数前加上第二个 fun 函数的声明或将它放到 g() 函数的前面则程序的运行结果会不同，显然第二个 fun 函数更符合。

#include <iostream>

void fun(int)
{
   std::cout << "int fun is called\n"; 
}
// void fun(double);
void g()
{
    fun(3.3);
}

void fun(double)
{
    std::cout << "double fun is called\n";
}

int main()
{
    g(); // int fun is called
}

#include <iostream>

namespace MyNS
{

    }

重载解析（重载决议）

重载解析：在名称查找的基础上进一步选择合适的调用函数

  1. 过滤不能被调用的版本 (non-viable candidates)

  2. 参数个数不对

  3. 无法将实参转换为形参
  4. 实参不满足形参的限制条件
#include <iostream>

void fun(int x)
{  
}
void fun(std::string x)
{  
}

int main()
{
    fun(3);
}

  1. 在剩余版本中查找与调用表达式最匹配的版本,匹配级别越低越好(有特殊规则)

  2. 级别 1 ：完美匹配 或 平凡转换(比如加一个 const )

  3. 级别 2 ： promotion 或 promotion 加平凡转换
  4. 级别 3 ：标准转换 或 标准转换加平凡转换
  5. 级别 4* ：自定义转换 或 自定义转换加平凡转换 或 自定义转换加标准转换
  6. 级别 5* ：形参为省略号的版本 void fun(...)
  7. 函数包含多个形参时,所选函数的所有形参的匹配级别都要优于或等于其它函数
